## Result summary

See [`tools/distill/data/compare-v1-v2-2026-05-09.md`](../tools/distill/data/compare-v1-v2-2026-05-09.md)
for the full 3-way comparison.

**Headline:** neither v2a nor v2b is a clean win over v1. Launch
decision: **ship v1 as planned for Tuesday HN/X/LinkedIn launch.**

- **v2a** (data scale only, qwen3.6-plus teacher): marginally better
  on strategic prompts (+0.20 markers vs v1) but materially regresses
  OOD leakage (3.75 vs v1's 1.50). Adapter is overfitting to strategic
  shape.
- **v2b** (DeepSeek v4-pro teacher): worse on strategic (-0.47 markers)
  but materially better on OOD (0.75 vs v1's 1.50). DeepSeek's
  stylistic register pulled the model away from Hammerstein's voice
  on framework-shaped prompts (Hammerstein's pre-flight prediction
  landed) but kept it more conservative on off-domain prompts.
- **v1** has the best balance of strategic + OOD behavior.

## v3 candidate experiments

The v2 results sharpen the v3 design space:

1. **v3a — mixed-mode v2a + ~10-15% off-domain instruction data.**
   Tests whether the strategic gain holds while OOD regression is
   prevented. Cost: ~$15 OpenRouter (extra data gen) + $1.20 RunPod.
   This is the natural follow-up to v2a's pattern.

2. **v3b — hybrid teacher (50% qwen3.6-plus, 50% DeepSeek v4-pro).**
   Tests Hammerstein's "v2.1 hybrid teacher" suggestion. Captures
   v2b's OOD benefit while keeping qwen's strategic register.
   Cost: ~$25 OpenRouter + $1.20 RunPod.

3. **v3c — base model bump to Qwen2.5-14B with v2a data.** Tests
   whether 7B is the ceiling. Cost: ~$5 RunPod (larger model = more
   compute) + reuse v2a data ($0).

Order of priority: v3a (cheapest, addresses load-bearing OOD issue),
then v3b (validates hybrid-teacher path), then v3c (base scaling).

## Decision gates

- **Δ ≥ +0.10 over v1 by Sunday evening:** swap v1→v2 in Tue HN
  launch artifact (HuggingFace + GGUF + model card update).
- **Δ ∈ [-0.03, +0.10]:** ship v2 as post-launch follow-up; v1 is
  the launch artifact.
- **Δ < -0.03:** revert; investigate (single-variable v2 means data
  scaling itself didn't help — likely teacher quality plateau or
  eval-set saturation).

## v3 candidates (only if v2 wins)

- v3a: teacher swap to DeepSeek v4 pro (variable: teacher).

- v3b: base model bump to Qwen2.5-14B (variable: base size).
